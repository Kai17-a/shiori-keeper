# ルートとフロー

## ルート

| Route           | Purpose                                            |
| --------------- | -------------------------------------------------- |
| `/`             | ダッシュボードの概要と最新ブックマーク             |
| `/bookmarks`    | ブックマーク一覧、作成、編集、削除、検索           |
| `/folders`      | フォルダ一覧と作成                                 |
| `/folders/[id]` | フォルダ詳細、配下ブックマーク一覧、名前変更、削除 |
| `/tags`         | タグ一覧と作成                                     |
| `/tags/[id]`    | タグ詳細、関連ブックマーク一覧、名前変更、削除     |
| `/settings`     | API 接続先の状態確認と疎通チェック                 |

## ユーザーフロー

### ダッシュボード

- 集計値と最新ブックマークのプレビューを表示する
- ブックマーク、フォルダ、タグの画面へ遷移できる
- サイドバーの子要素を描画する前にカタログデータを読み込む

### ブックマーク

- URL、タイトル、任意の説明、フォルダ、タグ付きでブックマークを作成する
- モーダルでブックマークを編集する
- 確認後にブックマークを削除する
- タイトルまたは URL で検索する
- フォルダとタグで絞り込む
- カードまたはリンクから外部 URL を開く

### フォルダ

- 一覧ページからフォルダを作成する
- モーダルでフォルダ名を編集する
- フォルダ詳細ページを開く
- 確認後にフォルダを削除する
- 選択フォルダに属するブックマークを表示する

### タグ

- 一覧ページからタグを作成する
- モーダルでタグ名を編集する
- タグ詳細ページを開く
- 確認後にタグを削除する
- 選択タグに関連するブックマークを表示する

### 設定

- runtime config の `apiBaseUrl` を参照する
- 接続状態を表示する
- `/health` に対して疎通確認を行う
- 解決済みのエンドポイント URL をユーザーに表示する

## 共通操作

- 主要なナビゲーションはサイドバーを使う
- CRUD 操作はカードとモーダルを使う
- フォルダ/タグ更新後はサイドバー用データを再取得して同期を保つ
