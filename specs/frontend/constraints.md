# 制約

## API 依存

- フロントエンドは `GET /settings` に依存してバックエンドの base URL を解決する。
- `API_BASE_URL` が未設定のとき、フロントエンドはアクセス中のホスト名を使って `http(s)://<host>:8000` を既定値として組み立てる。
- `useBookmarkApi()` は設定取得結果を保持し、解決済み base URL を全 API リクエストに付与する。
- リクエストエラーは `extractErrorMessage()` で正規化する。

## UI 構造

- アプリは default レイアウトで共通のダッシュボードシェルを使う。
- フォルダとタグの子ナビゲーションはサイドバーカタログから生成されるため、古いカタログ状態はツリー表示に影響する。
- `BookmarkCard` は一覧表示側で補完された `folder_name` を前提にする。

## 状態とデータ

- ブックマークの作成・編集フォームは、フォルダとタグの関連付けを任意で受け付ける。
- ブックマーク一覧ページは、サーバー側フィルタに加えてクライアント側検索文字列も扱う。
- フォルダ/タグ詳細ページは、親エンティティと関連ブックマークを両方読み込む。

## UX ルール

- 破壊的操作は確認モーダルを必須とする。
- 保存フローは成功・失敗ともに toast で通知する。
- 設定画面は、設定済みの base URL と接続失敗の両方を表示できる必要がある。
- API が利用できない場合でも UI は落とさず、エラー状態を表示する。

## スタイリング

- アプリは Nuxt UI のダッシュボードコンポーネントと共通の `font-sans` テーマを使う。
- レイアウトはレスポンシブで、モバイルとデスクトップの両方で動作する必要がある。
- サイドバー幅の固定や、デスクトップ専用の操作前提に依存しない。
