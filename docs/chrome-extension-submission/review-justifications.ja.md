# 審査用説明

## プライバシー

Shiori Keeper は、ブックマークの保存と管理という拡張機能の主な機能を提供するために必要な権限のみを使用します。

ユーザーがブックマークを保存する際に現在のタブのタイトルと URL を読み取り、`chrome.storage.local` に API サーバー URL を保存し、既存の保存項目を URL ベースで確認・管理するためにブックマーク関連のアクセスを使用します。閲覧履歴を収集することはなく、広告配信やトラッキング目的でも使用しません。また、ユーザーが設定した Shiori Keeper API サーバーへブックマークデータを送信する以外に、第三者へユーザーデータを共有しません。

Chrome ウェブストアの審査においては、拡張機能の中核機能に必要な最小限のデータアクセスに限定することで、Chrome ウェブストア デベロッパー プログラム ポリシーを遵守しています。

## 単一用途

Shiori Keeper の用途は 1 つだけです。それは、ユーザーが現在閲覧しているページを Shiori Keeper アプリにブックマークとして保存し、ユーザーが立てた Shiori Keeper サーバーに送信し、その後その保存内容を管理することです。

拡張機能の機能はすべてこの目的を支えるためのものです。

- アクティブなタブのタイトルと URL を取得する
- 設定済みの Shiori Keeper API サーバーへブックマークデータを送信する
- API サーバー URL を拡張機能のローカルストレージに保存する
- 保存前にサーバー接続を確認する
- 必要に応じて既存ブックマークを URL で読み取り、更新する

拡張機能には用途と無関係な機能は含まれておらず、一般的なブラウジング、広告、解析、バックグラウンドでのデータ収集を目的としていません。

## 権限

Shiori Keeper のブラウザ拡張機能は、ユーザーが閲覧中のページを素早くブックマークとして保存し、既存の保存内容を確認・更新できるようにするために、以下の権限を必要とします。

### `tabs`

現在開いているタブの情報を取得するために必要です。

拡張機能のポップアップから、アクティブなタブの `title` と `url` を読み取り、そのページをブックマークとして保存します。ユーザーが手入力で URL をコピーしなくても、今見ているページをそのまま登録できるようにするための権限です。

### `storage`

拡張機能の設定をユーザーごとに保存するために必要です。

主に API サーバーの接続先 URL を `chrome.storage.local` に保持し、ブラウザを再起動した後も同じ設定を使えるようにします。これにより、毎回設定を入力し直さずに拡張機能を利用できます。

### `bookmarks`

Chrome のブックマーク機能と連携するために必要です。

Shiori Keeper では、既存ブックマークの確認や更新、URL を基準にした重複管理を行います。`bookmarks` 権限を使うことで、Chrome 上のブックマーク情報を参照し、ユーザーが保存したリンクを一貫して管理できるようにします。

### 補足

この拡張機能は、権限を要求する目的を上記の機能提供に限定しています。
広告表示、トラッキング、ユーザーの閲覧履歴収集など、機能と無関係な用途には使用しません。

## host_permissions

Shiori Keeper が `host_permissions` を要求するのは、ユーザーが選択した Shiori Keeper API サーバーと通信するためです。

拡張機能は、ブックマークデータの送信、API サーバーへの接続確認、フォルダやタグの読み込み、ブックマークの更新や削除を、設定済みのサーバー経由で行います。サーバー URL はユーザーが自由に設定でき、拡張機能はユーザーがセルフホストで立てた環境で使うことを想定しているため、ユーザーが設定した API エンドポイントへアクセスできる host 権限が必要です。

この権限は上記の API 通信にのみ使用し、無関係な閲覧処理やバックグラウンド通信には使用しません。
